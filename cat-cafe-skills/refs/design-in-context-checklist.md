# Design-in-Context Checklist (在地设计检查单)

凡是往已有页面/组件添加新 UI 元素的 feature，在 Design Gate 必须确认以下内容：

- [ ] 读过目标页面的实际组件代码（标注文件路径 + 关键行号）
- [ ] 列出该区域已有的 UI 元素（按钮、指示器、空间占用），或附上当前页面真实结构截图
- [ ] 明确新能力是新增入口、替代旧入口，还是与旧入口共存？
- [ ] 明确新元素放置位置及原因，并给出至少一个备选位置/形态和 trade-off
- [ ] 评估密度上限，说明移动端/窄宽度的退化策略（隐藏？折叠？移位？）
- [ ] 明确现有 UX 是变好还是变差，为什么？
- [ ] 覆盖不同状态：空态、已有态、切换/变更时的即时反馈
- [ ] 若跨 surface（如 Hub / Connector / Sidebar），说明如何保持一致
- [ ] [视觉专属] 视觉冲突检查：新元素的颜色、阴影、层级（z-index）是否与周围元素打架？是否复用了现有的 Design Token？
